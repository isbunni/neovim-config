## Overview

A clean, modular Neovim configuration written in Lua — focused on C/C++, Lua, and web development.

**Key features:**
- 🎨 Four themes with live switching (gruvbox, onedark, tokyonight, catppuccin)
- 🔭 Telescope fuzzy finding
- 🌳 Treesitter syntax highlighting
- 🔧 LSP with Mason (auto-installs `lua_ls` and `clangd`)
- ✏️ Auto-formatting via none-ls (prettier, clang-format, stylua)
- 🦘 Harpoon for quick file navigation
- 🐛 DAP debugging for C/C++ via GDB
- 📊 Git signs, auto-pairing, undo tree, dashboard

## Prerequisites

| Dependency | Why | Install |
|---|---|---|
| **Neovim ≥ 0.10** | Required for all features | `sudo pacman -S neovim` or [neovim.io](https://neovim.io) |
| **Git** | Plugin manager pulls from git | `sudo pacman -S git` |
| **GCC / Clang** | Treesitter parsers need a C compiler | `sudo pacman -S gcc` |
| **Node.js + npm** | Required for LSP servers and formatting | `sudo pacman -S nodejs npm` |
| **GDB** | Debugging (optional) | `sudo pacman -S gdb` |
| **A Nerd Font** | Icons in file tree and dashboard | [nerdfonts.com](https://www.nerdfonts.com/font-downloads) |
| **`~/.local/bin/theme`** | Theme switcher (syncs nvim + kitty + starship) | [See below](#theme-switcher) |

### Theme Switcher (Optional)

The `<leader>tt` theme switcher calls a shell script at `~/.local/bin/theme` that syncs your colors across Neovim, Kitty, and Starship.

If you **don't** use all three, you can still use the config — just don't worry about the switcher. The theme will be set on next nvim launch regardless.

## Installation

### Fresh Install (no existing config)

```bash
cd ~/.config
git clone https://github.com/isbunni/neovim-config.git nvim
```

### Migrating from Another Config

**Option A — Replace entirely:**

```bash
# Back up your old config
mv ~/.config/nvim ~/.config/nvim.bak

# Clone fresh
git clone https://github.com/isbunni/neovim-config.git ~/.config/nvim
```

**Option B — Install on top of existing:**

```bash
# Back up your old config first
cp -r ~/.config/nvim ~/.config/nvim.bak

# Clone into a temp location and copy just the Lua config
git clone https://github.com/isbunni/neovim-config.git /tmp/nvim-migrate
cp -r /tmp/nvim-migrate/init.lua ~/.config/nvim/init.lua
cp -r /tmp/nvim-migrate/lua/b ~/.config/nvim/lua/b
```

## First Launch

1. Open Neovim:
   ```bash
   nvim
   ```

2. Lazy.nvim will **automatically bootstrap**, clone all plugin repos, and install them.

3. Install Treesitter parsers (happens automatically on first open via `:TSUpdate`).

4. Install LSP servers — in Neovim, run:
   ```
   :Mason
   ```
   This installs `lua_ls` and `clangd`.

5. Restart Neovim. Everything should work.

## Plugin Layout

```
init.lua                    → Entry point
lua/b/
  options.lua               → vim.opt settings (line numbers, tabs, etc.)
  keymaps.lua               → All keybindings
  lazy.lua                  → Plugin manager bootstrap + loader
  theme.lua                 → Auto-generated by theme switcher — do not edit
  plugins/
    colorschemes.lua        → Theme definitions
    treesitter.lua          → Syntax highlighting config
    telescope.lua           → Fuzzy finder
    lsp.lua                 → Mason, LSP, formatting, completion
    dap.lua                 → C/C++ debugging with GDB
    editor.lua              → File tree, git signs, autopair, harpoon, dashboard, undo tree
```

## Updating Plugins

Inside Neovim:
```
:Lazy update
```
Or from the terminal:
```bash
nvim --headless "+Lazy! update" +qa
```

---

## Adding a Plugin

Edit `lua/b/plugins/` and add a new file or append to an existing one:

```lua
-- lua/b/plugins/mytool.lua
return {
  "author/plugin-name",
  config = function()
    require("plugin").setup({ })
  end,
}
```

No need to touch any other file — Lazy auto-discovers everything in `lua/b/plugins/`.
